## The terminal was never the product

DalkkakAI started as "a native multi-pane terminal for running several startups at once." That framing kept getting commoditized — every new first-party agent feature eats a little more of "run agents in parallel." The reframe that survives: the terminal is the *substrate* where work happens and data is born; the product is the **connective layer** above it — a living graph of every connection point across the whole portfolio.

The six core layers (terminal, multi-session, Claude Code augmentation, viz, skills, cross-startup dashboard + advisor) aren't six features. They're bound into one operating system by that shared spine: layer 3 *feeds* it, layers 4 / 6 / 6b *read* it.

## How the graph stays true without busywork

Two buckets. **External metrics** (billing, uptime) are *pulled* from real sources. **Internal change** — user flows, architecture, issues, "각종 변경" — is *pushed* by the agents in a platform-standard format, **as a byproduct of work**. The hard constraint, stated plainly by Jason: logging and standards must never distort the work. More tokens is fine; bending the task to fit the log is not. The work is done thoroughly on its own terms, and the record is supplementary, by format. That's what ends the era of hand-recording every change.

The standard itself: fixed and versioned (so the graph stays joinable), human-first markdown (defined in whatever direction a *person* finds most comfortable), refined by dogfooding, platform-owned, and every node carries provenance — confirmed / inferred / hypothesis — so an inferred node can never pass as fact.

## The shape of it

Mission, refined: a tool a solo founder needs to **start → build → manage → operate** new services — several at once. That four-stage lifecycle is the organizing axis, mapped cleanly onto the layers (start = scaffold, build = terminal + Claude Code, manage = the issue/structure/flow graph, operate = dashboard + advisor). Security follows the same instinct: consumer-appropriate — per-project path scoping, explicit confirmed grants for anything more, never a scary OS-wide permission ask.

Written into `BLUEPRINT.md` §1 (mission) and new §5.5 (The Connective Layer). This is the canonical blueprint now.
